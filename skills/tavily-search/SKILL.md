# Tavily Search 技能

## 用法

当智能体需要实时信息时，触发此技能：

### 触发条件
- 用户询问"最新股价"、"实时数据"、"现在多少"
- 用户搜索"最新新闻"、"今天发生了什么"
- 智能体需要获取实时 API 文档
- 智能体需要验证信息（如版本号、更新状态）

### 使用流程

1. **构造搜索查询**
   ```
   将用户的问题转化为 Tavily 兼容的搜索查询
   ```

2. **调用 Tavily API**
   ```
   使用 Tavily Search API 执行搜索
   ```

3. **处理搜索结果**
   ```
   解析返回的结果，提取关键信息
   ```

4. **整合到回答中**
   ```
   将搜索到的实时信息整合到智能体的回答中
   ```

### 限制
- ⚠️ 需要 Tavily API Key
- ⚠️ 有每日查询限制
- ⚠️ 响应时间取决于网络状况

### 示例

**示例 1：查询实时股价**
```
用户问："腾讯现在多少钱？"
→ 触发 tavily-search
→ 搜索："腾讯控股 实时股价 2026"
→ 整合："根据最新数据，腾讯控股当前价格为 533.00 港元"
```

**示例 2：搜索最新新闻**
```
用户问："今天有什么重要新闻？"
→ 触发 tavily-search
→ 搜索："2026年2月17日 重要新闻 科技"
→ 整合："根据最新报道，今天科技领域的重要新闻包括..."
```

**示例 3：查询 API 文档**
```
智能体需要："查询 OpenClaw 最新 API 文档"
→ 触发 tavily-search
→ 搜索："OpenClaw API 文档 最新版本 2026"
→ 整合："根据 OpenClaw 官方文档，最新版本的 API 变化包括..."
```

## 配置

### API 配置
```bash
# 设置 Tavily API Key
export TAVILY_API_KEY="your-api-key-here"
```

### 环境要求
- ✅ 网络连接
- ✅ 有效的 Tavily API Key
- ✅ Python 3.7+（如果使用 Python 脚本）

### 安全注意事项
- ⚠️ 不要在代码中硬编码 API Key
- ⚠️ 敏感信息（如 API Key）应存储在环境变量中
- ⚠️ 遵守 Tavily API 使用条款

## 工作流程

### 智能体调用流程
1. 智能体检测到需要实时信息
2. 智能体决定使用 tavily-search 技能
